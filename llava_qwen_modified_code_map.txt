LLaVA/Qwen Profiling Framework Modified Code Map
===============================================

목적
----
이 문서는 "원래 TinyEngine 기본 코드"가 아니라, 이번 작업에서 LLaVA/Qwen host-side profiling 및
STM32N6570-DK bring-up 준비를 위해 추가하거나 수정한 코드의 위치와 역할만 정리한 문서이다.

3) TinyEngine/src/kernels/fp_backward_op/llm_ops_fp.c
- 역할:
  LLaVA/Qwen 스타일 decoder/operator path에 필요한 FP32 baseline operator 구현.
- 이번 작업에서 한 일:
  TinyEngine에 없던 LLM/VLM용 공통 operator를 새로 구현했다.
- 구현된 함수:

  - gelu_fp
  - qkv_attention_fp
- 성격:

4) TinyEngine/src/kernels/fp_backward_op/qwen_block_fp.c
- 역할:
  Qwen decoder block 1개를 실행하는 wrapper.
- 이번 작업에서 한 일:
  TinyEngine operator를 조합해 "Qwen block microbenchmark"용 wrapper를 구현했다.
- 실행 순서:
  1. RMSNorm
  2. Q Linear
  3. K Linear
  4. V Linear
  5. RoPE
  6. repeat_kv
  7. GQA attention
  8. O Linear
  9. residual add
  10. RMSNorm
  11. gate Linear
  12. up Linear
  13. SiLU
  14. gated multiply
  15. down Linear
  16. residual add
- 추가 포인트:
  - workspace layout 계산 포함
  - qwen_block_fp_get_workspace_floats / bytes 제공
  - profiling 목적의 standalone FP32 linear baseline 포함


5) TinyEngine/src/kernels/fp_backward_op/mm_projector_fp.c
- 역할:
  LLaVA mm_projector path wrapper.
- 이번 작업에서 한 일:
  projector를 Linear -> GELU -> Linear 형태로 구현했다.
- 주요 API:
  - mm_projector_fp_get_workspace_floats
  - mm_projector_fp_get_workspace_bytes
  - mm_projector_fp_run


6) TinyEngine/src/kernels/fp_backward_op/lm_head_last_token_fp.c

- 용도:

7) TinyEngine/src/kernels/fp_backward_op/kv_cache_fp.c

- 이번 작업에서 한 일:

- 주요 API:

  - kv_cache_fp_get_cache_bytes
  - kv_cache_append_fp
  - kv_cache_read_fp


8) TinyEngine/src/kernels/fp_backward_op/image_text_fusion_fp.c

3. STM32N6570-DK Bring-Up 준비 코드
----------------------------------

12) examples/llava_microbench/benchmark_timer.h
- 역할:
  host / STM32 공통 timer interface 헤더.
- 이번 작업에서 한 일:
  timer abstraction API를 정의했다.
- API:
  - benchmark_timer_init
  - benchmark_timer_now
  - benchmark_timer_elapsed_cycles
  - benchmark_timer_elapsed_ms


13) examples/llava_microbench/benchmark_timer_stm32.c
- 역할:
  STM32 timer implementation.
- 이번 작업에서 한 일:
  STM32N6570-DK bring-up용 timer wrapper를 구현했다.
- 동작 방식:
  1. DWT cycle counter 사용 가능하면 우선 사용
  2. DWT가 안 되면 HAL_GetTick fallback
  3. host syntax check 환경에서는 clock() fallback
- 특징:
  - HAL header가 없는 host build에서도 깨지지 않도록 guarded fallback 적용
  - SystemCoreClock이 있으면 cycles -> ms 변환에 사용


14) examples/llava_microbench/main_stm32n6.c
- 역할:
  STM32N6570-DK bring-up 전용 toy microbenchmark entry.
- 이번 작업에서 한 일:
  host benchmark와 분리된 STM32 전용 entry를 만들었다.
- 실행 benchmark:
  - rmsnorm_fp
  - gelu_fp
  - silu_fp
  - mm_projector_fp
  - image_text_fusion_fp
  - qwen_block_fp
  - lm_head_last_token_fp
- toy config:
  - seq_len = 4
  - hidden_size = 32
  - intermediate_size = 64
  - num_heads = 4
  - num_kv_heads = 2
  - rotary_dim = 8
  - head_dim = 8
  - vision_hidden = 32
  - num_image_tokens = 4
  - text_tokens = 4
  - vocab_size = 128
  - runs = 10
- 중요한 정책:
  - malloc/free 사용 안 함
  - 모든 large buffer를 static/global로 둠
  - workspace bytes를 먼저 계산하고 capacity를 넘으면 alloc_fail 출력
  - checksum과 NaN/Inf 여부 출력
  - CSV 형식은 host와 최대한 유사하게 유지


4. Memory Planning Tool / Config Sweep Tool
-----------------------------------------

15) tools/estimate_llava_mcu_memory.py
- 역할:
  reduced LLaVA-style config의 memory를 host에서 계산하는 estimator.
- 이번 작업에서 한 일:
  다음 입력을 받아 memory breakdown을 계산하는 스크립트를 만들었다.
  - vision_params
  - vision_tokens
  - vision_hidden
  - decoder_layers
  - hidden_size
  - num_heads
  - num_kv_heads
  - mlp_ratio
  - text_tokens
  - vocab_size
  - dtype_bytes
- 출력 항목:
  - vision_weight_MB
  - projector_weight_MB
  - decoder_weight_MB
  - embedding_weight_MB
  - lm_head_weight_MB
  - kv_cache_MB
  - activation_MB
  - attention_score_MB
  - total_weight_MB
  - total_runtime_MB
- 용도:
  MCU-friendly shrink 방향을 host에서 먼저 추정.


16) tools/sweep_llava_mcu_configs.py
- 역할:
  MCU 축소 후보 config sweep 및 feasible filtering.
- 이번 작업에서 한 일:
  search space를 순회하며 estimator를 재사용해 memory를 계산하고,
  budget 조건으로 feasible 여부를 판단해 CSV 파일을 생성하는 스크립트를 만들었다.
- 생성 파일:
  - results/mcu_config_sweep.csv
  - results/mcu_config_feasible.csv
- reason column:
  - weight_over_budget
  - kv_cache_over_budget
  - activation_over_budget
  - runtime_over_budget

9. 현재 이 코드들의 의미
----------------------

지금까지 추가한 코드는 "원본 LLaVA 0.5B 전체를 STM32에 올리는 코드"가 아니다.

현재 의미는 아래와 같다.

1. host에서 LLaVA/Qwen 스타일 operator path를 FP32 기준으로 재현 가능하게 함
2. Qwen block, projector, lm_head, kv_cache, image-text fusion을 독립 benchmark 가능하게 함
3. MCU-friendly shrink 후보를 memory estimator와 config sweep으로 사전에 걸러냄
4. STM32N6570-DK에서 먼저 toy config로 build / flash / printf / timer / checksum / no hard fault를 확인할 수 있게 bring-up entry를 준비함

즉, 이 수정들은 "최종 모델 포팅"이 아니라 "포팅 전 준비 + profiling framework + MCU shrink planning"에 해당한다.
